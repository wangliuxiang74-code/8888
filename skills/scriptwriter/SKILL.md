# SKILL.md - 编剧创作

## Description
剧本创作、人物设定、分集大纲、对话编写。

## When to Use
- 新漫剧立项
- 剧本修改
- 人设补充
- 情节设计

## Workflow

### 1. 类型定位
**言情向：**
- 甜宠（高甜无虐）
- 虐恋（玻璃渣里找糖）
- 先婚后爱
- 霸总/千金

**玄幻向：**
- 修仙升级
- 穿越重生
- 系统流
- 甜宠玄幻

### 2. 人物设定
**主角设定：**
- 姓名、年龄、外貌
- 性格特点（3个核心标签）
- 背景故事（原生家庭/经历）
- 目标/动机（想要什么）
- 弱点/恐惧（增加真实感）

**配角设计：**
- 反派（有理由的坏）
- 闺蜜/兄弟（助攻型）
- 工具人（推动剧情）

### 3. 剧情结构
**3集结构（短视频）：**
- 第1集：冲突引入（钩子）
- 第2集：矛盾升级（虐点）
- 第3集：反转/甜宠（ payoff）

**长剧结构（50-100集）：**
- 起：人物登场，问题出现（1-10集）
- 承：感情发展，障碍出现（11-40集）
- 转：大冲突，误会/分手（41-80集）
- 合：问题解决，HE（81-100集）

### 4. 剧本格式
```
场景：客厅 - 日
人物：女主、男主

【画面】女主坐在沙发上，眼眶微红
女主（OS）：三年了，他还是忘不掉她...

【画面】男主推门而入，看到女主，愣住
男主：你怎么还没睡？

【画面】女主抬头，强忍泪水
女主：我等你。
```

### 5. 爽点/虐点设计
**爽点：**
- 打脸反派
- 男主护妻
- 身份揭露
- 误会解除

**虐点：**
- 误会错过
- 被迫分离
- 牺牲成全
- 时间差（错过）

### 6. 平台适配
**抖音：**
- 3集一个完整小剧情
- 节奏快，反转多
- 评论区引导追更

**B站：**
- 可以长一点，5-10分钟
- 弹幕互动点设计
- 二创空间预留

**小红书：**
- 图文+视频结合
- 美图优先
- 笔记引流

## Tools
- 飞书文档（协作）
- 编剧软件（Final Draft）
- AI辅助（ChatGPT脑暴）

## Notes
- 每集必须有钩子（让人想看下集）
- 人设不能崩，OOC是大忌
- 狗血可以，但要合理
- 数据不好及时调整剧情
